# nstbrowser-ai-agent - Quick Reference

## Prerequisites

**Required**: Nstbrowser desktop client must be installed and running
- Download: https://www.nstbrowser.io/
- Start the desktop application
- Obtain API key from dashboard

## Quick Setup

```bash
# 1. Configure API key (required)
nstbrowser-ai-agent config set key YOUR_API_KEY

# 2. Verify connection
nstbrowser-ai-agent profile list

# 3. Ready to use!
```

## Core Commands

### Configuration
```bash
nstbrowser-ai-agent config set key YOUR_KEY    # Set API key
nstbrowser-ai-agent config show                # Show current config
```

### Profile Management
```bash
nstbrowser-ai-agent profile list               # List all profiles
nstbrowser-ai-agent profile create <name>      # Create new profile
nstbrowser-ai-agent profile delete <id>        # Delete profile
```

### Browser Control
```bash
nstbrowser-ai-agent browser start-once         # Start temporary browser
nstbrowser-ai-agent browser list               # List running browsers
nstbrowser-ai-agent browser stop <id>          # Stop browser
nstbrowser-ai-agent --profile <name> open <url> # Use specific profile
```

### Page Interaction
```bash
nstbrowser-ai-agent open <url>                 # Navigate to URL
nstbrowser-ai-agent snapshot -i                # Get interactive elements with refs
nstbrowser-ai-agent click @e1                  # Click element by ref
nstbrowser-ai-agent fill @e2 "text"           # Fill input field
nstbrowser-ai-agent get text @e3              # Extract text content
nstbrowser-ai-agent screenshot [path]          # Take screenshot
nstbrowser-ai-agent close                      # Close browser
```

### Data Extraction
```bash
nstbrowser-ai-agent get title                  # Get page title
nstbrowser-ai-agent get url                    # Get current URL
nstbrowser-ai-agent eval "document.title"      # Execute JavaScript
```

## Quick Start Workflows

### Workflow 1: Temporary Browser
```bash
# Start temporary browser and navigate
nstbrowser-ai-agent browser start-once
nstbrowser-ai-agent open https://example.com
nstbrowser-ai-agent snapshot -i
nstbrowser-ai-agent click @e1
nstbrowser-ai-agent close
```

### Workflow 2: Profile-based Session
```bash
# Use persistent profile
nstbrowser-ai-agent --profile my-profile open https://example.com
nstbrowser-ai-agent fill @e1 "user@example.com"
nstbrowser-ai-agent fill @e2 "password"
nstbrowser-ai-agent click @e3
nstbrowser-ai-agent close
```

### Workflow 3: Data Extraction
```bash
# Extract information from page
nstbrowser-ai-agent open https://example.com
nstbrowser-ai-agent snapshot -i
nstbrowser-ai-agent get text @e1
nstbrowser-ai-agent eval "document.querySelectorAll('h2').length"
nstbrowser-ai-agent screenshot page.png
```

## JSON Output

Add `--json` to any command for machine-readable output:
```bash
nstbrowser-ai-agent profile list --json
nstbrowser-ai-agent snapshot -i --json
nstbrowser-ai-agent get text @e1 --json
```

## Common Issues & Solutions

| Problem | Solution |
|---------|----------|
| "HTTP 400: Bad Request" | Start Nstbrowser desktop client |
| "Profile not found" | Use `profile list` to see available profiles |
| "Element not found" | Run `snapshot -i` to refresh element refs |
| "@e1 not working" | Element refs change - get fresh snapshot |

## Advanced Features

- **Proxy Configuration**: `profile proxy update <id> --host proxy.com --port 8080`
- **Batch Operations**: `profile delete id1 id2 id3`
- **Tab Management**: `tab new <url>`, `tab 0`, `tab close`
- **Wait Strategies**: `wait 3000`, `wait --load networkidle`

## Best Practices

1. **Use Profile IDs**: More reliable than names for browser operations
2. **Refresh Refs**: Run `snapshot -i` after tab switches or page changes
3. **Use JSON Output**: Add `--json` for AI agent integration
4. **Handle Errors**: Check command output and retry if needed
5. **Close Cleanly**: Always run `close` to save session state

## Notes

- All browser operations use Nstbrowser profiles for advanced fingerprinting
- Temporary browsers (`browser start-once`) don't save session state
- Profile sessions persist cookies, localStorage, and login state
- Element refs (@e1, @e2) are generated by `snapshot -i` command
